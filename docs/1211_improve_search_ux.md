# 검색 경험 개선 및 중복 표시 해결

## 1. 개요
사용자 피드백을 반영하여 단축어 검색 시 안내 메시지를 제공하고, 검색 결과에서 동일 학과가 반복 표시되는 문제를 해결했습니다.

## 2. 변경 내용

### 2.1 단축어 검색 가이드 (UX 개선)
- **문제**: "컴공", "경영" 등 짧은 단어로 검색 시 유사도 검색 품질이 저하될 수 있음.
- **해결**: 검색어 길이가 3글자 이하인 경우, 결과 하단에 Tip 메시지를 노출.
  > 💡 Tip: '컴공' 같은 줄임말보다는 '컴퓨터공학과'처럼 정식 명칭으로 검색하면 더 정확한 결과를 찾을 수 있어요!

### 2.2 검색 결과 중복 제거 (Display Logic)
- **문제**: "컴퓨터공학과" 검색 시 결과 목록에 동일한 학과가 여러 번 나타남.
  - 원인: Vector DB가 문서를 Chunk 단위(요약, 진로, 교과목 등)로 저장하므로, 유사도 검색 시 여러 Chunk가 동시에 매칭됨.
- **해결**: 검색 결과를 사용자에게 보여줄 때 **학과 ID(`major_id`)를 기준으로 중복을 필터링(Deduplication)**하여, 상위 1개의 대표 결과만 노출되도록 개선.

## 3. 결과 확인
- `backend/scripts/test_major_similarity.py` 실행 시 더 이상 중복된 학과가 나오지 않습니다.
- `list_departments("컴공")` 호출 시 하단에 안내 문구가 표시됩니다.

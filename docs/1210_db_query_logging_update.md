# DB 쿼리 로깅 기능 추가

**작성일**: 2025-12-10
**상태**: 완료

---

## 1. 개요
기존 로깅 시스템은 애플리케이션 레벨의 로그(`unigo_app`)만 수집하고 있어, 데이터베이스 쿼리 수행 내용(SQL)을 확인하기 어려웠습니다. 디버깅 및 성능 분석을 위해 실행되는 SQL 쿼리를 로깅하도록 기능을 확장했습니다.

## 2. 수정 내용

### 2.1 `settings.py` 로깅 설정 추가
Django의 내장 로거인 `django.db.backends`를 활성화하여 SQL 쿼리가 로그 파일에 기록되도록 설정했습니다.

*   **수정 파일**: `unigo/settings.py`
*   **추가된 로거**:
    ```python
    "django.db.backends": {
        "handlers": ["file", "console"],
        "level": "DEBUG",
    },
    ```
    *   `DEBUG` 레벨로 설정하여 모든 쿼리 내역을 수집합니다.
    *   기존 `file` 핸들러와 `console` 핸들러를 공유하여 `unigo.log` 파일과 콘솔 양쪽에 출력됩니다.

## 3. 트러블슈팅 및 해결 과정

### 3.1 로그 레벨 고려
*   **문제**: 기본적으로 `INFO` 레벨에서는 SQL 쿼리가 출력되지 않음.
*   **해결**: `django.db.backends` 로거의 레벨을 `DEBUG`로 명시적으로 설정하여 쿼리 로그가 누락되지 않도록 했습니다.

## 4. 결과
이제 `logs/unigo.log` 파일에서 애플리케이션 로그뿐만 아니라 실행된 SQL 쿼리문도 확인할 수 있어, DB 상호작용 분석이 용이해졌습니다.

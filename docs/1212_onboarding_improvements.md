# 온보딩 질문 개선 (2025-12-12)

## 개요
전공 추천 정확도를 높이기 위해 사용자로부터 수집하는 온보딩 질문을 수정했습니다.
기존 질문들은 사용자의 단순 흥미나 희망 연봉에 초점이 맞춰져 있었으나, DB에 저장된 전공 데이터의 특성(진로, 적성 등)을 충분히 활용하지 못하는 한계가 있었습니다.

이를 개선하기 위해 데이터 기반으로 질문을 재구성함으로서, Vector Search(의미 기반 검색)의 품질을 향상시켰습니다.

## 변경 사항

### 1. 질문 변경 상세

| 기존 질문 | 변경/추가 질문 | 변경 사유 (데이터 기반) |
|:---:|:---:|:---|
| **선호 고교 과목** | **선호 교과목** (유지/보완) | DB의 `relate_subject` 필드(관련 교과목)와 직접 매칭됨. |
| **흥미 및 취미** | **관심사 및 활동** (보완) | 단순 취미를 넘어 동아리 등 관련 활동(`career_act`) 정보를 수집하여 매칭 정확도 향상. |
| **희망 연봉** | **장래 희망** (신설/교체) | DB의 `job`(관련 직업) 및 `enter_field`(진출 분야) 필드는 매우 상세한 텍스트로 구성되어 있어, 연봉 숫자보다 검색 정확도가 월등히 높음 (가중치 1.2). |
| (없음) | **성격 및 장점** (신설) | DB의 `interest`(적성 및 흥미) 및 `property`(학과 특성) 필드에 포함된 "논리적 사고", "상상력" 등의 키워드와 매칭하기 위함. |
| **희망 학과** | **희망 학과** (유지) | 기존의 점수 부스팅(Tier 1~3) 로직 유지를 위해 필수. |

### 2. 제거된 질문
- **희망 연봉**: 고등학생 입장에서 현실적인 연봉을 제시하기 어렵고, 실제 DB 매칭 시 단순 숫자(Salary)보다는 직업명(Job)이 더 명확한 검색 기준이 되기 때문에 **장래 희망** 질문으로 대체되었습니다.

## 기대 효과
- **`Career Goal` (장래 희망)**: "인공지능 개발자"라고 답할 경우, 전공 설명에 포함된 직업명과 직접 매칭되어 추천 정확도 급상승.
- **`Strengths` (성격/장점)**: "꼼꼼하다", "논리적이다" 등의 답변이 전공별 적성 설명(`interest`)과 매칭되어, 단순히 과목 성적만으로는 알 수 없는 적성 기반 추천 가능.

## 코드 변경
- **Frontend**: `unigo/static/js/chat.js`의 `ONBOARDING_QUESTIONS` 배열 수정.
- **Backend compatibility**: `backend/graph/nodes.py`의 `_build_user_profile_text` 함수는 이미 `career_goal`, `strengths` 키를 처리할 수 있도록 구현되어 있어 별도 수정 불필요.

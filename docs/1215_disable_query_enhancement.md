# 자동 쿼리 확장 중단

## 개요
사용자의 구체적인 단일 학과명 질문(예: "컴퓨터공학과 남녀 성비", "경영학과 등록금")이 시스템 내부 로직에 의해 "해당 학과에 대한 포괄적 소개 요청"으로 자동 확장되는 문제를 해결했습니다.

## 문제 원인
`backend/graph/nodes.py`의 `agent_node`에서 사용자 질문이 단일 학과 관련 키워드를 포함하고 길이가 짧을 경우(`helper.py`의 `is_single_major_query`), 이를 자동으로 풍부한 질문으로 변환(`enhance_single_major_query`)하는 로직이 강제 적용되고 있었습니다. 이로 인해 사용자가 특정 정보만을 원해도 전체적인 정보를 제공하려다 보니 원치 않는 긴 답변이나 불필요한 툴 호출이 발생했습니다.

## 해결 방법
- `backend/graph/nodes.py`에서 `is_single_major_query` 및 `enhance_single_major_query`를 호출하는 코드 블록을 주석 처리하여 비활성화했습니다.
- 이제 사용자의 질문은 변형 없이 그대로 LLM에게 전달되며, LLM은 질문의 문맥에 맞춰 필요한 정보만 답변하게 됩니다.

## 결과
- "컴퓨터공학과 남녀 성비" 질문 시, "컴퓨터공학과에 대해 자세히 알려주세요..."로 변환되지 않고 성비 관련 정보만 답변합니다.
- 불필요한 대학 목록 조회(list_departments 등) 빈도가 감소했습니다.

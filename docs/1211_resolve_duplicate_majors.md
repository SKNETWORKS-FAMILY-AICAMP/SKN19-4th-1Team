# 학과 데이터 중복 문제 해결 (Vector DB)

## 1. 개요
Vector DB(Pinecone)에 "컴퓨터공학과" 중복(`major-179-0`) 및 기타 잔존 데이터(`major-0-0` 등)가 존재하여 해결하였습니다.

## 2. 원인 분석
- 과거 버전 코드에서 생성된 `major-` prefix ID를 가진 데이터들이 인덱스 초기화 과정에서 완전히 제거되지 않고 일부 남아있었습니다 (Stale Data).

## 3. 조치 내용
1. **인덱스 재구축**: 최신 데이터를 재적재하여 정상 데이터 확보.
2. **개별 강제 삭제**: 검색에 노출되던 `major-179-0` 강제 삭제.
3. **일괄 삭제 (Complete Cleanup)**: 
   - 스크립트(`backend/scripts/delete_stale_majors.py`)를 통해 `major-`로 시작하는 모든 ID를 스캔하여 일괄 삭제했습니다.

## 4. 결과
- "컴퓨터공학과" 검색 시 중복 없음 확인.
- 전체 DB 스캔 결과 `major-` 패턴의 잔존 데이터 **0건** 확인.
